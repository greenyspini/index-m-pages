# index-m-pages

一个用于归档和发布多个独立 HTML 页面的仓库。本项目不针对任何特定域名或网站，仅作为静态页面的集中存放与展示空间。

## 项目简介

index-m-pages 旨在以简单、直接的方式保存和发布独立的 HTML 页面。这些页面可能包括信息展示页、工具界面、临时存档页或任何适合以单文件形式呈现的内容。仓库内的页面彼此独立，不依赖外部数据库或后端服务，方便快速浏览与分享。

## 目录说明

```
.
├── pages/          # 存放所有独立 HTML 页面文件
├── assets/         # 页面所需的静态资源（如 CSS、JS、图片等）
├── index.html      # 可选：仓库根目录的引导页或索引页
└── README.md       # 本文件
```

- `pages/`：核心目录，每个子文件夹或文件对应一个独立页面。
- `assets/`：公共资源目录，供多个页面复用。
- `index.html`：可选的入口页面，用于快速导航至仓库内各个页面。

## 页面归档说明

每个页面均为完整的 HTML 文件，自包含样式与脚本。归档时遵循以下原则：

- 页面文件名应清晰反映内容，避免使用无意义编号。
- 若页面依赖外部资源（如字体、库），建议内联或一并存放于 `assets/` 目录。
- 页面间无硬性关联，可独立打开运行。

## 维护说明

- 欢迎通过 Pull Request 提交新的页面，或对现有页面进行改进。
- 提交前请确保页面在主流浏览器中正常显示，无外部失效链接。
- 避免在页面中包含广告、跟踪代码或恶意脚本。
- 本仓库仅供学习、归档与个人使用，请勿用于违反法律法规或平台政策的内容。
